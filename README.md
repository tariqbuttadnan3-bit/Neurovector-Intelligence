# NeuroVector Intelligence

## Overview
**NeuroVector Intelligence** is a Java-based real-time strategic simulation system that demonstrates the practical use of Data Structures and Algorithms (DSA) through an interactive grid-based decision environment. It simulates intelligent agents operating in a dynamic system where every action is driven by optimized algorithms, prioritization, and predictive decision-making.

The project focuses on shortest-path navigation, real-time event processing, and adaptive behavior analysis to create a structured simulation of competitive strategy and optimization.

---

## Objective
The main objective of this project is to apply core DSA concepts in a practical and interactive system by:

* Simulating real-time decision-making processes  
* Implementing efficient pathfinding using graph algorithms  
* Managing system events using queue-based processing  
* Prioritizing actions using heap-based structures  
* Tracking performance and analytics using optimized data structures  

---

## Key Features

### *Real-Time Simulation Engine*
Processes movement, actions, and interactions continuously in a dynamic grid-based environment.

---

### *Intelligent Pathfinding System*
Uses graph traversal algorithms to compute:

* Shortest paths  
* Optimal escape routes  
* Efficient target navigation  

---

### *Priority-Based Decision System*
Implements PriorityQueue to ensure that the most important actions are executed first in real time.

---

### *Event Processing System*
Uses Queue (FIFO structure) to manage and execute simulation events in an ordered and systematic way.

---

### *Replay & History Tracking*
Uses Stack to store and replay previous actions, enabling undo functionality and simulation analysis.

---

### *Performance Analytics Module*
Tracks and evaluates:

* Decision speed  
* Movement efficiency  
* Survival performance  
* Action patterns  

using HashMap and sorting techniques.

---

## Data Structures & Algorithms Used

* *ArrayList* → Dynamic storage of entities and players  
* *LinkedList / Queue* → Event processing (FIFO execution)  
* *Stack* → Action history and replay system  
* *HashMap* → Fast lookup of statistics and simulation data  
* *PriorityQueue* → Priority-based decision engine  
* *Graphs* → Grid-based world representation  
* *BFS (Breadth First Search)* → Shortest path calculation  
* *DFS (Depth First Search)* → Exploration and traversal logic  
* *Sorting Algorithms* → Leaderboard and ranking system  

---

## System Modules

### *Simulation Arena Module*
Represents the environment as a grid-based graph where all entities interact and move.

---

### *Event Management Module*
Handles all actions using queue-based processing to ensure correct execution order.

---

### *Decision Engine Module*
Uses priority-based logic to determine optimal actions for AI agents in real time.

### *Pathfinding Module*
